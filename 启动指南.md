# 职业规划智能体 - 项目启动指南

## 一、环境准备

- **Python 3.x**（用于前端静态服务 + AI 算法服务）
- 当前前端为静态 HTML，无需 Node.js

## 二、启动步骤

### 方式 A：使用脚本启动（推荐）

1. **启动前端**
   - 双击运行项目根目录下的：`start-frontend.bat`
   - 或在命令行执行：
     ```bash
     cd d:\Projects\Service-Outsourcing-Innovation-and-Entrepreneurship-Competition
     start-frontend.bat
     ```
   - 前端将运行在 **http://localhost:8888**
   - 保持该窗口打开，勿关闭

2. **启动 AI 服务**（可选，职业测评、职业规划报告等 AI 功能需要）
   - 双击运行：`start-ai.bat`
   - 或在命令行执行：
     ```bash
     cd d:\Projects\Service-Outsourcing-Innovation-and-Entrepreneurship-Competition
     start-ai.bat
     ```
   - AI 服务将运行在 **http://127.0.0.1:5001**
   - 保持该窗口打开，勿关闭

3. **访问系统**
   - 在浏览器打开：**http://localhost:8888**
   - 即可进入登录页

---

### 方式 B：手动命令行启动

打开两个命令行窗口：

**终端 1 - 前端：**

## 三、登录说明

当前为**模拟模式**（`api.js` 中 `mockMode: true`），支持任意账号密码登录：

| 登录方式 | 账号 | 密码 | 说明 |
|----------|------|------|------|
| 预设账号 | `admin` | `123456` | 管理员（默认预填） |
| 预设账号 | `test` / `demo` / `student` | `123456` | 演示用账号 |
| **自定义** | 任意（≥2 位） | 任意（≥6 位） | 输入自己的账号和密码即可登录 |

> 模拟模式下，其他人可输入自己的账号和密码登录，系统会自动创建虚拟用户身份，无需事先注册。数据保存在浏览器本地，关闭页面后仍可再次用相同账号密码登录。

## 四、服务端口说明

| 服务     | 端口 | 说明                         |
|----------|------|------------------------------|
| 前端     | 8888 | 静态页面，登录、测评、岗位画像、职业规划等 |
| AI 算法  | 5001 | 职业测评、岗位画像、职业规划报告       |

> **提示**：默认 `mockMode: true`，无需启动 Java 后端即可使用。登录、档案、推荐等使用本地模拟数据。若需真实 AI 功能（测评、职业规划报告生成），将 `api.js` 中 `mockMode` 改为 `false` 并启动 AI 服务。

## 五、常见问题

1. **端口被占用**
   - 修改 `start-frontend.bat` 中的端口（如改为 3000、8080）
   - 或关闭占用该端口的其他程序

2. **登录失败**
   - 账号至少 2 位、密码至少 6 位
   - 点击「登录异常？清除缓存重试」后重试
   - 强制刷新页面（Ctrl+F5）清除缓存

3. **页面打不开**
   - 确认前端服务已启动（终端无报错）
   - 尝试访问 http://127.0.0.1:8888

4. **职业测评/职业规划报告无响应**
   - 确认已启动 AI 服务（`start-ai.bat` 或方式 B 终端 2）

5. **连接不上后端 / 登录失败 / 接口报错**
   - 打开 `frontfiles/api.js`，将 `mockMode` 改为 `true`
   - 模拟模式下无需启动 Java 或 AI 服务，登录、档案、岗位匹配等均使用本地模拟数据
   - 需要真实 AI 功能时：改回 `mockMode: false`，并确保 AI 服务（5001）已启动
